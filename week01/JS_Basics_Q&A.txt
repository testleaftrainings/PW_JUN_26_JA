1. String Immutability
--------------------------------------------------------------------------------

Q: "In JavaScript, strings are described as immutable. Can you explain what that 
   means in practical terms? When we talk about 'modifying' a string, what 
   actually happens behind the scenes?"

A: 
   • Strings are immutable → cannot be changed after creation
   • Any modification creates a NEW string in a new memory location
   • The variable reference is updated to point to the new value
   • Original string remains unchanged

   Example:
   let str = "test";        // memory A
   str = str + "leaf";      // memory B (new string created)
   console.log(str); // testleaf


2. Extract ID from String
--------------------------------------------------------------------------------

Q: "Suppose you have a dynamic string like this:
   'Order ID: #12345-Confirmed'
   Extract only the numeric ID."

A:
   • Use split() to break string using delimiters
   • Extract value between # and -

   Example:
   let text = "Order ID: #12345-Confirmed";
   let id = text.split("#")[1].split("-")[0];
   console.log(id); // 12345


3. Handling Falsy Value (0 Bug)
--------------------------------------------------------------------------------

Q: "Consider this validation function:
   function isValid(data) {
       if (data) { return true; }
   }
   The intention is to check whether a field has a value. However, there's a bug: 
   the number 0 is being treated as invalid (falsy), but 0 is actually a valid 
   input for a quantity field. How would you rewrite this validation logic?"

A:
   • if(data) fails because 0 is falsy
   • Use explicit checks for null, undefined, and empty string

   Example:
   function isValid(data) {
     return data !== null && data !== undefined && data !== "";
   }
   console.log(isValid(0)); // true


4. Modifying String Characters
--------------------------------------------------------------------------------

Q: "I have the following code:
   let str = 'Hello World';
   str[0] = 'J';
   What will happen when I try to change the first character this way?
   Also, since strings are immutable, how does JavaScript actually handle 
   string storage in memory? And what is the correct way to achieve a change 
   like replacing 'H' with 'J'?"

A:
   • Strings are immutable → direct index assignment does NOT work
   • JavaScript ignores the change silently
   • Must create a new string

   Example:
   let str = "Hello World";
   str = "J" + str.slice(1);
   console.log(str); // Jello World


5. Parsing Server Logs
--------------------------------------------------------------------------------

Q: "You're given a continuous string of server logs:
   ERROR: 404 [Time: 12:00] | INFO: Login Success [Time: 12:01] | ERROR: 500 [Time: 12:05]
   Your task is to extract an array containing only the timestamps (like 12:00 
   and 12:05) for log entries that contain 'ERROR'."

A:
   • Split logs using |
   • Filter entries containing "ERROR"
   • Extract time using string methods

   Example:
   let logs = "ERROR: 404 [Time: 12:00] | INFO: Login Success [Time: 12:01] | ERROR: 500 [Time: 12:05]";
   
   let result = logs
     .split("|")
     .filter(log => log.includes("ERROR"))
     .map(log => log.split("[Time: ")[1].replace("]", "").trim());
   
   console.log(result); // ["12:00", "12:05"]


6. Longest Word in Sentence
--------------------------------------------------------------------------------

Q: "Take this sentence:
   'Automation testing is incredibly important'
   Write a loop — or use any approach you prefer — to find and return the 
   longest word in the sentence."

A:
   • Split sentence into words
   • Loop and compare lengths
   • Track longest word

   Example:
   let sentence = "Automation testing is incredibly important";
   
   let words = sentence.split(" ");
   let longest = "";
   
   for (let word of words) {
     if (word.length > longest.length) {
       longest = word;
     }
   }
   
   console.log(longest); // incredibly


7. replaceAll() in Test Automation
--------------------------------------------------------------------------------

Q: "In the context of test automation, can you give me an example or scenario 
   where using replaceAll() would be the right choice? What problem does it 
   solve that replace() might not handle properly?"

A:
   • Used for data cleanup and normalization
   • Replaces all occurrences in a string
   • Useful for:
     - Removing symbols
     - Cleaning API/UI data
     - Preparing data for assertions

   Example:
   let price = "$1,000.50";
   let clean = price.replaceAll(/[^0-9.]/g, "");
   console.log(clean); // 1000.50


8. Remove Duplicates from String
--------------------------------------------------------------------------------

Q: "Suppose you have a string with duplicate characters, like 'aabbcdeeff'.
   How would you write logic to remove the duplicates and return a string with 
   only unique characters, preserving the original order?"

A:
   • Use Set (removes duplicates, preserves order)
   • Convert back to string

   Example:
   let str = "aabbcdeeff";
   let result = [...new Set(str)].join("");
   console.log(result); // abcdef
